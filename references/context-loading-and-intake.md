# Context Loading And Intake

Use alongside `using-aw-skills` and `aw-plan`.

## Loading Order

1. repo-local contracts and rules
2. approved AW artifacts
3. baseline profiles, platform docs, and `.aw_rules`
4. source files and diffs
5. logs, traces, screenshots, and failing commands
6. conversation history

## Intake Questions

- what stage is this actually in
- what evidence already exists
- what org-standard playbooks apply
- what is the smallest correct next action

## Anti-Flood Rule

Do not load every related file "just in case."
Load only what changes the current stage decision.
