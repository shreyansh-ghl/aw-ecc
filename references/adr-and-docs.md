# ADR And Docs

Use when the change affects architecture, rollout expectations, or long-lived team knowledge.

## Document The Why

- architecture changes
- interface compatibility decisions
- rollout and rollback constraints
- unusual quality-gate or operational exceptions

## Good ADR Shape

- context
- decision
- alternatives considered
- consequences

## GHL Notes

- cross-team platform decisions need stronger rationale because they affect shared playbooks
- launch-specific caveats belong in release artifacts when they affect shipping behavior
